(देखना है कोई इसका हिन्दी में अनुवाद करता है कि नहीं， एक विदेशी को भेजा था＝ )
Ellipse Geometry:
Vertex (V): The point on the ellipse farthest from the center along the major axis.
Center ©: The midpoint of the line segment joining the two foci.
Focus (F): One of the two fixed points inside the ellipse.
Distances:
VC (b): The distance from the vertex to the center, which is the semi-minor axis.
VF ©: The distance from the vertex to the focus.
FC (a): The distance from the focus to the center, which is the semi-major axis.
Relationship:
The relationship between these distances is given by: [ c^2 = a^2 - b^2 ]
This equation shows that the distance from the center to the focus © is derived from the semi-major axis (a) and the semi-minor axis (b).
Angles:
Angle FVC: The angle formed by the lines VF and VC. This angle is half of the maximum Equation of Center (EOC).
Angle F1VF2: The angle formed by the lines from one focus (F1) to the vertex (V) and then to the other focus (F2). This angle represents the maximum EOC (Equation of Center = Manda phala).

Let’s break down the key points:
Sheeghraphala and Mandaphala:
Sheeghraphala: This correction is heliocentric, meaning it is based on the Sun’s position relative to the planet.
Mandaphala: This correction is the Equation of the Center (EOC), which accounts for the elliptical shape of the planet’s orbit.
Sheeghraphalaardha and Mandaphalaardha:
Sheeghraphalaardha: Half of the Sheeghraphala.
Mandaphalaardha: Half of the Mandaphala (EOC).
Transformation from Ellipse to Geocentric:
The Suryasiddhantic system involves transforming the calculations from the center of the ellipse to a geocentric (Earth-centered) perspective.
The process involves adding Sheeghraphalaardha to the mean planet position to get Paragraha, then adding Mandaphalaardha to Paragraha to get Dvigraha.
Crucial Point:
The full Mandaphala is subtracted from Mandakendra to compute the next Mandaphala, but the full Mandaphala is added to the Mean Planet, not to Dvigraha.
This indicates that the Suryasiddhantic orbits of the five planets are not simple ellipses. The first two half-phalas (Sheeghraphalaardha and Mandaphalaardha) define a shift from the ellipse’s focus.

Learning to use AVX and SSE instructions can significantly boost the performance of your applications, especially when dealing with large datasets. Here are some resources and steps to get you started:
Getting Started with AVX
Understand the Basics: AVX (Advanced Vector Extensions) are used to perform single instruction on multiple data (SIMD) operations. This can greatly speed up tasks like matrix multiplication, signal processing, and more.
Enable AVX: Ensure AVX is enabled on your system. You can check and enable it through your BIOS settings. Here’s a guide on how to do that.
Programming with AVX: You can use AVX instructions in your code using intrinsic functions provided by your compiler. These functions allow you to write code that directly uses AVX instructions without needing to write assembly code. This example explains how to use AVX in C++.
Getting Started with SSE
Understand the Basics: SSE (Streaming SIMD Extensions) is another set of SIMD instructions that can be used for similar tasks as AVX but is generally used for smaller datasets.
Programming with SSE: Like AVX, you can use intrinsic functions to utilize SSE instructions in your code. This example provides an introduction to SSE instructions and how to use them.
Example Code
Here’s a simple example of how you might use AVX intrinsics in C++ to add two arrays of floats. You can find more detailed tutorials and examples online to help you get started with more complex operations.
#include <immintrin.h>
#include <iostream>
void add_arrays(float* a, float* b, float* result, int size) {
for (int i = 0; i < size; i += 8 ) {
__m256 vec_a = _mm256_loadu_ps(&a[i]);
__m256 vec_b = _mm256_loadu_ps(&b[i]);
__m256 vec_result = _mm256_add_ps(vec_a, vec_b);
_mm256_storeu_ps(&result[i], vec_result);
}
}
int main() {
const int size = 16;
float a[size] = { /* initialize with values */ };
float b[size] = { /* initialize with values */ };
float result[size];
add_arrays(a, b, result, size);
for (int i = 0; i < size; ++i) {
std::cout << result[i] << " ";
}
return 0;
}

शीघ्रफल: यह सुधार सूर्यकेंद्रित है, जिसका अर्थ है कि यह ग्रह के सापेक्ष सूर्य की स्थिति पर आधारित है।
मंदफल: यह सुधार केंद्र का समीकरण (ईओटी) है, जो ग्रह की कक्षा के अण्डाकार आकार को दर्शाता है।
शीघ्रफलार्ध और मन्दफलार्ध: शीघ्रफलार्ध: शीघ्रफल का आधा भाग।
मंदफलार्ध: मंदफल (ईओटी) का आधा भाग।
दीर्घवृत्त से भूकेंद्रिक में परिवर्तन: सूर्यसिद्धांतिक प्रणाली में गणनाओं को दीर्घवृत्त के केंद्र से भूकेंद्रिक (पृथ्वी-केंद्रित) परिप्रेक्ष्य में परिवर्तित करना शामिल है।
इस प्रक्रिया में पाराग्रह प्राप्त करने के लिए माध्य ग्रह स्थिति में शीघ्रफलार्ध को जोड़ना, फिर द्विग्रह प्राप्त करने के लिए पाराग्रह में मंदफलार्ध को जोड़ना शामिल है।

आणविक भौतिकी में हाइजेनबर्ग के “मैट्रिकस मैकेनिक्स” को एक शती बीत गयी । उनके युग में कम्प्यूटर नहीं थे । अब SSE (Streaming SIMD Extensions) AVX (Advanced Vector Extensions) प्रायः सभी पर्सनल कम्प्यूटरों में है । मेरे कम्प्यूटरों में AVX2 के CPU हैं जो न्यूनतम 512x512 तथा अधिकतम 1024x1024 elements के मैट्रिक्सों का गणित की क्षमता रखते हैं और उनमें छोटे मैट्रिक्सों के लिए SSE के सारे वर्सन भी हैं ।
AVX और SSE निर्देशों का उपयोग करना सीखना आपके अनुप्रयोगों के प्रदर्शन को महत्वपूर्ण रूप से बढ़ा सकता है, खासकर जब बड़े डेटासेट से निपटना हो। आरंभ करने के लिए यहां कुछ संसाधन और चरण दिए गए हैं:
AVX के साथ आरंभ करना
मूल बातें समझें: AVX (उन्नत वेक्टर एक्सटेंशन) का उपयोग कई डेटा (SIMD) संचालन पर एकल निर्देश निष्पादित करने के लिए किया जाता है। यह मैट्रिक्स गुणन, सिग्नल प्रोसेसिंग और अन्य जैसे कार्यों को बहुत तेज़ कर सकता है।
AVX सक्षम करें: सुनिश्चित करें कि आपके सिस्टम पर AVX सक्षम है। आप अपनी BIOS सेटिंग्स के माध्यम से इसे जाँच और सक्षम कर सकते हैं। ऐसा करने के तरीके के बारे में यहाँ एक मार्गदर्शिका दी गई है।
AVX के साथ प्रोग्रामिंग: आप अपने कंपाइलर द्वारा प्रदान किए गए आंतरिक फ़ंक्शन का उपयोग करके अपने कोड में AVX निर्देशों का उपयोग कर सकते हैं। ये फ़ंक्शन आपको असेंबली कोड लिखने की आवश्यकता के बिना सीधे AVX निर्देशों का उपयोग करने वाला कोड लिखने की अनुमति देते हैं। यह वीडियो बताता है कि C++ में AVX का उपयोग कैसे करें।
SSE के साथ आरंभ करना
मूल बातें समझें: SSE (स्ट्रीमिंग SIMD एक्सटेंशन) SIMD निर्देशों का एक और सेट है जिसका उपयोग AVX जैसे ही कार्यों के लिए किया जा सकता है, लेकिन आम तौर पर इसका उपयोग छोटे डेटासेट के लिए किया जाता है।
SSE के साथ प्रोग्रामिंग: AVX की तरह, आप अपने कोड में SSE निर्देशों का उपयोग करने के लिए आंतरिक फ़ंक्शन का उपयोग कर सकते हैं। यह वीडियो SSE निर्देशों और उनका उपयोग करने के तरीके का परिचय प्रदान करता है।
उदाहरण कोड
यहाँ एक सरल उदाहरण दिया गया है कि आप C++ में AVX इंट्रिन्सिक्स का उपयोग दो ऐरे जोड़ने के लिए कैसे कर सकते हैं। यह उदाहरण दर्शाता है कि फ़्लोट की दो ऐरे जोड़ने के लिए AVX इंट्रिन्सिक्स का उपयोग कैसे करें। आप अधिक जटिल ऑपरेशन के साथ आरंभ करने में सहायता के लिए ऑनलाइन अधिक विस्तृत ट्यूटोरियल और उदाहरण पा सकते हैं।
#include <immintrin.h>
#include <iostream>
void add_arrays(float* a, float* b, float* result, int size) {
for (int i = 0; i < size; i += 8 ) {
__m256 vec_a = _mm256_loadu_ps(&a[i]);
__m256 vec_b = _mm256_loadu_ps(&b[i]);
__m256 vec_result = _mm256_add_ps(vec_a, vec_b);
_mm256_storeu_ps(&result[i], vec_result);
}
}
int main() {
const int size = 16;
float a[size] = { /* initialize with values */ };
float b[size] = { /* initialize with values */ };
float result[size];
add_arrays(a, b, result, size);
for (int i = 0; i < size; ++i) {
std::cout << result[i] << " ";
}
return 0;
}

दीर्घवृत्त ज्यामिति:
शीर्ष (V): दीर्घवृत्त पर केंद्र से सबसे दूर स्थित बिंदु।
केंद्र ©: दो फोकस को जोड़ने वाले रेखाखंड का मध्यबिंदु।
फोकस (F): दीर्घवृत्त के अंदर दो स्थिर बिंदुओं में से एक।
दूरियाँ:
VC (b): शीर्ष से केंद्र तक की दूरी, जो अर्ध-लघु अक्ष है।
VF ©: शीर्ष से फोकस तक की दूरी।
FC (a): फोकस से केंद्र तक की दूरी, जो अर्ध-लघु अक्ष है।
संबंध:
इन दूरियों के बीच संबंध इस प्रकार दिया गया है: [c^2 = a^2 - b^2]
यह समीकरण दर्शाता है कि केंद्र से फोकस © की दूरी अर्ध-लघु अक्ष (a) और अर्ध-लघु अक्ष (b) से प्राप्त होती है।
कोण:
कोण FVC: रेखाओं VF और VC द्वारा बनाया गया कोण। यह कोण अधिकतम केंद्र समीकरण (EOC) का आधा है।
कोण F1VF2: एक फोकस (F1) से शीर्ष (V) और फिर दूसरे फोकस (F2) तक रेखाओं द्वारा बनाया गया कोण। यह कोण अधिकतम EOC (समय का समीकरण = मंदफल) को दर्शाता है।
सूर्यसिद्धांत पर अपनी टिप्पणी में, बर्गेस ने कहा कि शीघ्रफल सूर्यकेंद्रीय > भूकेंद्रीय है, और मंदफल ईओटी है। लेकिन उन्होंने कहा कि तीसरे सुधार शीघ्र-फलार्ध(शीघ्रफल का आधा) और मंदफलार्ध (ईओटी का आधा) के पीछे का तर्क समझ से परे है। मंदफलार्ध का अर्थ है कि सूर्यसिद्धांतिक प्रणाली दीर्घवृत्त के केंद्र से भूकेंद्रीय में परिवर्तित हो जाती है। लेकिन सूर्यसिद्धांतिक प्रणाली इतनी सरल नहीं है।
माध्य ग्रह + शीघ्रफलार्ध > परग्रह, फिर परग्रह + मंडपफलार्ध > द्विग्रह।
अगले मंदफल की गणना करने के लिए इस द्विग्रह को मंदकेंद्र से घटाया जाता है, लेकिन उस पूरे मंदफल को द्विग्रह में नहीं बल्कि माध्य ग्रह में जोड़ा जाता है: यह सबसे महत्वपूर्ण बिंदु है। इसका मतलब दीर्घवृत्तीय कक्षा में केवल तीसरा (पूर्ण मंदफल) और चौथा (पूर्ण शीघ्रफल) होगा, लेकिन पहले दो आधे फल दीर्घवृत्त के फोकस से कुछ बदलाव को परिभाषित करते हैं। बर्गेस ने इस बदलाव की गणना करने की कोशिश नहीं की। किसी और ने भी गणना करने की कोशिश नहीं की।
